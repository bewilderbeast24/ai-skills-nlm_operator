# Operation Handover

Use this format for final reports after NotebookLM CLI operations.

## Request

State the user request in one sentence.

## Selected Mode

State one of `notebooks`, `sources`, `tags`, `artifacts`, or `registry`.

## Commands

List the `nlm` commands that were run. Omit secrets, tokens, and private credential paths.

## Result

Summarize the CLI outcome and any user-visible NotebookLM state change.

## Registry Update

State the registry path, whether it changed, and the affected IDs or aliases.

## Validation

State whether registry validation passed. If validation did not run, state the reason.

## Follow-up

List only concrete unresolved blockers or user decisions required.
